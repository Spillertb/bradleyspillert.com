# Design Rules

## Visual Identity

- **Dark mode by default** — dark backgrounds (#0a0a0a / near-black), light text
- **Modern and clean** — generous whitespace, clear hierarchy, smooth transitions
- **NOT AI-generated looking** — no generic gradients, no stock-photo vibes, no cookie-cutter templates
- **Subtle personality** — this is a real person's site, not a corporate landing page

## Typography

- Clean sans-serif (Inter or system stack)
- Large hero text, comfortable body text (16-18px base)
- Use font weight for hierarchy, not excessive font sizes

## Colors

- Primary background: dark (#0a0a0a to #111)
- Text: white/light gray (#e5e5e5 for body, #fff for headings)
- Accent: subtle blue or teal for links/highlights — don't overdo it
- Use color sparingly — mostly monochrome with accent pops

## Components

- **Cards** — slight border or subtle background differentiation, rounded corners
- **Navigation** — sticky, minimal, active page highlighting
- **Buttons/Links** — clear hover states, smooth transitions
- **Skill pills** — small rounded tags for tech skills

## Animations

- Subtle and purposeful — no flashy entrances
- Smooth page transitions if possible
- Three.js globe should be interactive but not distracting

## Responsive

- Mobile-first approach
- Globe may be hidden or simplified on small screens
- Navigation collapses gracefully

## Hard No List

- ❌ No emojis anywhere on the site
- ❌ No dev jargon in user-facing copy ("leveraging synergies", "full-stack wizard")
- ❌ No generic stock phrases ("passionate about technology")
- ❌ No light mode (dark mode only for now)
- ❌ No excessive animations or parallax scrolling
- ❌ No popup modals or cookie banners
